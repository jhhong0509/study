# service
### 빈 주입 방식
- @Autowired
    - 의존 객체의 타입에 맞는 빈을 주입한다.
    - 의존성 주입이 간단하다.
    - 추천되지 않음
        - final이 불가능해 객체가 변할 수 있다.
        - 의존관계가 보이지 않아 추상적이고 의존성 관계가 복잡해진다.
- setter
    - 생성자에 모든 의존성을 기술하면 복잡해질 수 있지만, 선택적으로 나누게 해줌.
    - 추천되지 않음
        - final 선언 불가능
    - 3.x까지는 추천되던 방식
- 생성자
    - 추천되는 방법
        - 순환참조를 알아차릴 수 있다.
        - @Autowired를 사용하지 않아도 된다.
        - final 가능
    - 어쩔 수 없는 순환참조는 해결이 힘듬
        - 다른 주입 방법 사용
